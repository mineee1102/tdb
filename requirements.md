
# tdb - TUI Database Client

## 概要

ターミナル上で動作するTUI形式のデータベースクライアントツール。
TablePlusのような直感的な操作感をターミナルで実現し、Homebrewで配布する。

## 基本情報

| 項目 | 内容 |
|------|------|
| ツール名 | `tdb` |
| 実装言語 | Go |
| TUIライブラリ | [Bubble Tea](https://github.com/charmbracelet/bubbletea) + [Lip Gloss](https://github.com/charmbracelet/lipgloss) |
| 配布方法 | Homebrew Tap |
| ライセンス | MIT |
| 対応OS | macOS / Linux / Windows |

## 対応データベース

| DB | ドライバ | 優先度 |
|----|---------|--------|
| PostgreSQL | `pgx` | P0 |
| MySQL | `go-sql-driver/mysql` | P0 |
| SQLite | `modernc.org/sqlite`（CGOフリー） | P0 |
| SQL Server | `microsoft/go-mssqldb` | P1 |

## 機能要件

### 1. 接続管理

- **接続一覧画面**: 保存済みの接続をリスト表示（TablePlusのサイドバー風）
- **接続情報の保存**: `~/.config/tdb/connections.yaml` に保存
- **接続文字列対応**: DSN形式での接続指定もサポート
- **CLI引数での接続**: `tdb --dsn "postgres://user:pass@localhost:5432/mydb"` のようなワンライナー接続
- **SSH トンネル対応**: SSH経由でのDB接続をサポート
- **接続テスト**: 接続確認機能

### 2. テーブル一覧・ナビゲーション

- **左ペインにテーブル一覧**: スキーマ > テーブル の階層表示
- **テーブル種別表示**: テーブル / ビュー / マテリアライズドビュー をアイコンで区別
- **テーブル検索**: インクリメンタルサーチでテーブルを絞り込み
- **キーボードナビゲーション**: `j/k` or 矢印キーでテーブル選択、`Enter` でデータ表示

### 3. データ閲覧（Data Viewer）

- **テーブル形式表示**: 横スクロール・縦スクロール対応のグリッド表示
- **カラム幅の自動調整**: データ内容に基づいてカラム幅を自動調整
- **ページネーション**: 大量データは `LIMIT/OFFSET` で段階的に読み込み（デフォルト100行）
- **NULL値の可視化**: NULL値を視覚的に区別表示（例: グレーの `NULL`）
- **行番号表示**: 行番号を左端に表示
- **セル内改行対応**: 長いテキストは折り返し or 省略表示を切り替え可能

### 4. フィルタ・ソート

- **カラムソート**: カラムヘッダ上で `s` キーでASC/DESC/解除を切り替え
- **WHERE句フィルタ**: `/` キーでフィルタ入力モードに入り、条件を指定
- **クイックフィルタ**: 選択中のセル値で即座にフィルタリング
- **ソート・フィルタ状態表示**: ステータスバーに現在のソート・フィルタ条件を表示

### 5. データ編集（CRUD）

#### 行の追加（Create）

- `a` キーで新規行の入力フォームを表示
- デフォルト値の自動入力
- NOT NULL 制約のバリデーション

#### セル編集（Update）

- `Enter` or `e` キーでセル編集モードに入る
- インライン編集（TablePlus風）
- **変更のハイライト**: 未コミットの変更をセル背景色で表示
- **一括コミット**: `Ctrl+S` で変更を一括反映（BEGIN/COMMIT）
- **変更の破棄**: `Ctrl+Z` で未コミットの変更を破棄

#### 行の削除（Delete）

- `d` キーで行を削除マーク（即座には削除しない）
- 削除マークされた行は取り消し線 or 赤色で表示
- `Ctrl+S` で一括反映

#### 変更プレビュー

- コミット前に実行されるSQLをプレビュー表示
- ユーザーが確認してから実行

### 6. SQLエディタ

- **SQLエディタペイン**: `:` or `Ctrl+E` で SQLエディタに切り替え
- **シンタックスハイライト**: SQL キーワードをハイライト表示
- **複数行入力**: 複数行のSQLを入力可能
- **実行**: `Ctrl+Enter` で SQL を実行し、結果をデータビューアに表示
- **クエリ履歴**: 過去に実行したクエリの履歴を保存・再利用（`~/.config/tdb/history.yaml`）
- **自動補完**: テーブル名・カラム名の自動補完
- **複数クエリ対応**: セミコロン区切りで複数クエリを実行し、タブで結果を切り替え

### 7. DDL操作

- **テーブル構造表示**: `i` キーでテーブルのカラム定義・インデックス・外部キーを表示
- **テーブル作成**: TUIフォームでテーブルを作成
- **カラム追加/変更/削除**: テーブル構造画面からカラム操作
- **インデックス管理**: インデックスの作成・削除
- **DDLプレビュー**: 操作に対応するDDLを確認してから実行

### 8. エクスポート

- **CSV出力**: 現在表示中のデータをCSVでエクスポート
- **JSON出力**: 現在表示中のデータをJSONでエクスポート
- **SQL出力**: INSERT文としてエクスポート
- **クリップボードコピー**: 選択範囲をTSV形式でクリップボードにコピー

## 非機能要件

### UI/UX

- **レスポンシブレイアウト**: ターミナルサイズの変更に追従
- **カラーテーマ**: ダーク/ライトテーマの切り替え対応
- **ステータスバー**: 接続情報・行数・選択位置を常時表示
- **キーバインドヘルプ**: `?` キーでコンテキストに応じたキーバインド一覧を表示
- **通知**: 操作結果のフィードバックをステータスバーに表示（成功/エラー）
- **モーダルダイアログ**: 確認・入力が必要な場合はモーダルで表示
- 操作はnvimのキーバインドを参考に、キーボード中心の操作体系を設計
- 可能であれば、nvimの設定（init.lua）を読み込み、nvim環境と同じキーバインドを提供するオプションを用意

### パフォーマンス

- **起動時間**: 0.5秒以内に初期画面を表示
- **クエリ結果表示**: 100行のデータを0.3秒以内にレンダリング
- **メモリ効率**: 大量データを全件メモリに載せず、ストリーミング的に処理
- **非同期クエリ**: クエリ実行中もUI操作をブロックしない

### セキュリティ

- **パスワード保存**: OS のキーチェーン/シークレット管理に統合（macOS Keychain / Linux secret-service）
- **接続文字列のマスク**: ログ・画面上でパスワードをマスク表示
- **SSL/TLS対応**: DB接続時のSSL/TLS設定をサポート

## キーバインド設計（デフォルト）

| キー | コンテキスト | 動作 |
|------|------------|------|
| `Tab` | グローバル | ペイン切り替え（テーブル一覧 ↔ データビュー ↔ SQLエディタ） |
| `j` / `k` | テーブル一覧 / データビュー | 上下移動 |
| `h` / `l` | データビュー | 左右スクロール |
| `g` / `G` | データビュー | 先頭行 / 最終行 |
| `Enter` | テーブル一覧 | テーブルを開く |
| `Enter` / `e` | データビュー | セル編集 |
| `a` | データビュー | 行追加 |
| `d` | データビュー | 行削除マーク |
| `s` | データビュー（カラムヘッダ） | ソート切り替え |
| `/` | データビュー | フィルタ入力 |
| `Ctrl+S` | データビュー | 変更をコミット |
| `Ctrl+Z` | データビュー | 変更を破棄 |
| `Ctrl+E` | グローバル | SQLエディタ切り替え |
| `Ctrl+Enter` | SQLエディタ | SQL実行 |
| `i` | テーブル一覧 | テーブル構造表示 |
| `?` | グローバル | キーバインドヘルプ |
| `q` / `Ctrl+C` | グローバル | 終了 |
| `y` | データビュー | 選択セルをコピー |

## Homebrew 配布

### Tap リポジトリ

- リポジトリ: `github.com/<owner>/homebrew-tap`
- Formula: `tdb.rb`

### インストール方法

```bash
brew tap <owner>/tap
brew install tdb
```

### リリースフロー

1. `git tag vX.Y.Z` でバージョンタグを作成
2. GitHub Actions で `goreleaser` を実行
3. macOS (amd64/arm64) / Linux (amd64/arm64) のバイナリを生成
4. Homebrew Formula を自動更新

### goreleaser 設定

- クロスコンパイル: macOS (amd64, arm64), Linux (amd64, arm64), Windows (amd64)
- アーカイブ形式: `.tar.gz` (macOS/Linux), `.zip` (Windows)
- チェックサムファイル生成
- Homebrew Formula 自動更新

## 開発フェーズ

### Phase 1: 基盤（MVP）

- プロジェクトセットアップ（Go module, Bubble Tea）
- DB ドライバ抽象化（PostgreSQL のみ）
- 接続管理（CLI引数）
- テーブル一覧表示
- データ閲覧（読み取り専用）
- 基本的なキーボードナビゲーション

### Phase 2: 編集機能

- CRUD操作（追加・編集・削除）
- 変更のハイライト・一括コミット
- フィルタ・ソート
- SQLエディタ（基本）

### Phase 3: 複数DB対応

- MySQL ドライバ追加
- SQLite ドライバ追加
- SQL Server ドライバ追加
- 接続情報の保存・管理画面

### Phase 4: 高度な機能

- DDL操作
- エクスポート機能
- 自動補完
- クエリ履歴
- SSHトンネル

### Phase 5: 配布・品質

- goreleaser セットアップ
- Homebrew Tap 作成
- テスト整備
- ドキュメント整備
- カラーテーマ
